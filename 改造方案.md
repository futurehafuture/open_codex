# Open Codex App — 改造方案（待修改清单）

> 目标定位：**官方 Codex 引擎当发动机 + 自己做一个能用、能发布的桌面 App**。
> 引擎不 fork、不改 Rust 源码，通过官方 `@openai/codex-sdk` 和 `codex app-server`（JSON-RPC）驱动。

---

## 0. 核心结论（为什么要改）

原仓库（4 个文件、约 470 行）用 `node-pty` 起一个子进程跑官方 Codex 的**全屏交互式 TUI**，再把终端字节原样塞进网页 `<pre>`。这叫"刮终端画面"，做产品不可行：拿到的是带 ANSI 转义码、会重绘的画面，分不清哪句是回复、哪个是要审批的命令、哪个是文件 diff。

官方引擎其实自带两套**给 App 用的程序接口**（原作者完全没用）：

| 接口 | 是什么 | 用在哪个阶段 |
|---|---|---|
| `@openai/codex-sdk`（npm，已发布 `0.142.2`） | 启动 codex 引擎，走 JSONL 收**结构化事件**（消息/推理/命令/文件改动/todo） | M0 快速跑通 |
| `codex app-server`（JSON-RPC 2.0 over stdio） | VS Code 扩展同款接口，含**多会话 + 交互式审批 + 登录** | M2 起的完整控制面 |

> 已核实：`@openai/codex-sdk@0.142.2` 锁死依赖 `@openai/codex@0.142.2`（正好是本仓库用的版本），两者绑定升级；SDK 只支持**预设档位审批**，**交互式"批准这一条命令"必须走 app-server**。

## 3. 按里程碑的详细改法

### M0 — 接 SDK，把刮屏换成结构化渲染（先让它真跑通一次）

1. **依赖**：`npm install` → `npm run rebuild`（rebuild 是给 native 模块对齐 Electron ABI）。
2. **`src/main.js` 重写**：
   - 删掉 agent 相关的 `pty.spawn` / `agent:start` / `agent:prompt` / `agent:write` / `agent:dispose`（这套是刮屏，作废）。
   - `const { createEngine, resolveCodexCommand, checkAgent } = require('./main/engine')`。
   - 新增 IPC：
     - `engine:check` → `checkAgent(codexPath)`
     - `engine:start`（sessionId, cfg）→ `engine.startSession(...)`
     - `engine:prompt`（sessionId, text）→ `engine.sendPrompt(...)`
     - `engine:interrupt` / `engine:respond-approval`
   - 引擎回调 `onEvent` 里：`mainWindow.webContents.send('engine:event', event)`。
   - **保留** `terminal:*` 的 pty（终端面板要用），只是 agent 这条路改掉。
3. **`src/preload.js`**：`openCodex` 暴露 `startEngine / sendPrompt / interrupt / respondApproval / onEngineEvent / checkAgent`，保留 `chooseWorkspace` 和 `terminal*`。
4. **`src/renderer.html` + 新增 `src/renderer/ui/agentView.js`**：
   - 收到 `engine:event` 后按 `event.kind` / `event.item.type` 渲染：
     - `agent_message` → 气泡正文
     - `reasoning` → 折叠的"思考"块
     - `command_execution` → 命令 + 退出码 + 输出
     - `file_change` → 文件 diff 列表
     - `todo_list` → 勾选列表
   - 删掉现在 `agentOut.textContent += data` 的裸字节追加。
5. **验收**：在「设置」填 Base URL + API Key → 选工作目录 → 输一句话 → 界面出现**分类渲染**的真实回复。
   - 凭证不走登录：SDK 把 `baseUrl` 转成 `--config openai_base_url=...`，把 `apiKey` 写进 `CODEX_API_KEY` 环境变量喂给 `codex exec`（已核实自 `@openai/codex-sdk@0.142.2` dist）。
   - SDK 走 `codex exec`（非交互），默认 `approval_policy="never"` + `sandbox="workspace-write"`，否则它无法执行命令/改文件，M0 会退化成纯聊天。

### M1 — 持久化 + 真实项目/会话（干掉假侧边栏）

1. 新增 `src/main/store/db.js`：`better-sqlite3` 初始化，建表 `projects / threads / messages`（threads 存 `engine_thread_id` 用于 resume）。
2. 新增 `repositories.js`：projects/threads/messages 的 CRUD。
3. 新增 IPC `store:*`：listProjects / createProject / listThreads / createThread / listMessages / appendMessage。
4. `main.js` 的 `onEvent` 里把 item 落库（消息、命令、文件改动）。
5. **`src/renderer.html` 侧边栏**：删掉写死的 `claudecode解析 / 面试code / gui-agents / 剧本ai` 及假时间戳，改成 `src/renderer/ui/sidebar.js` 从 `store:listProjects` 渲染。
6. **验收**：新建项目/会话 → 重启 App 仍在；点历史会话能回看消息，并能 resume（用 `engine.startSession({resumeThreadId})`）。

### M2 — 切 app-server + 交互式审批 + 登录

1. **校准协议字段**：新增 `scripts/gen-protocol.js` 跑 `codex app-server generate-ts --out src/main/engine/generated`，按真实 schema 核对 `appServerAdapter.js` 里 `normalizeAppServerItem()` 的字段名（现在是防御式映射，先能跑、再精确）。
2. **`main.js`**：加引擎模式开关（`sdk` ↔ `app-server`），把 `kind:'approval_request'` 事件透传给渲染层；接 `engine:respond-approval`(requestId, decision)。
3. **新增 `src/renderer/ui/approvals.js`**：收到审批请求时弹窗，展示 `command/cwd/reason` 或 `changes`(diff)，按钮 → `openCodex.respondApproval(requestId, 'accept'|'acceptForSession'|'decline'|'cancel')`。
   - 协议细节（已确认）：server 主动发 `item/commandExecution/requestApproval` / `item/fileChange/requestApproval`，client 回 `{ decision }`，随后 `serverRequest/resolved` + `item/completed`。
4. ~~登录引导~~ **已取消**：改为设置式 Base URL/Key（M0 已实现：`src/main/config/store.js` + 设置弹窗）。app-server 也读 `CODEX_API_KEY`；但其 Base URL 覆盖方式尚未核实（见 `appServerAdapter.js` 注释），M2 再校准。
5. **验收**：app-server 模式下触发一条需审批的命令 → 弹窗 → 批准后继续，`item/completed` 显示结果。

### M3 — xterm 真终端 + 打包

1. **新增 `src/renderer/ui/terminalPanel.js`**：用 `@xterm/xterm` + `@xterm/addon-fit` 替换现在 `renderer.html` 里只能逐字符的 `keydown` 处理；接 `terminal:data` / `terminal:write` / `terminal:resize`。引入 `@xterm/xterm/css/xterm.css`。
2. **新增 `electron-builder.yml`**：`mac`（dmg、zip）、`win`（nsis、msi）target；`files` 包含 `src` 与 `node_modules`。
3. **原生模块**：`better-sqlite3` / `node-pty` 已配 `postinstall: electron-builder install-app-deps`。
4. **`scripts/check.js`**：替换 `check-html.js`，做全量 `node --check` + 关键 IPC 通道存在性检查。
5. **验收**：`npm run dist` 产出安装包；终端面板能跑 `vim` / `top` 等交互式程序。
